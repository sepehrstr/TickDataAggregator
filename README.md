# TickDataAggregator

Тестовое задание: система агрегации и обработки данных с биржевых торгов.

## Описание

Проект реализует упрощённую систему сбора, нормализации, дедупликации и сохранения тиковых данных из нескольких WebSocket-источников.

На входе система получает поток сырых сообщений от бирж в разных форматах. Далее данные проходят через единый pipeline:

1. получение сообщений из нескольких источников одновременно;
2. нормализация к единой модели;
3. дедупликация;
4. батчевое сохранение в PostgreSQL;
5. логирование и базовые метрики.

## Что было важно при реализации

При выполнении задания я сознательно старался **не переусложнять решение** и держаться в рамках требований тестового задания.

Поэтому в проекте нет избыточной распределённой инфраструктуры и тяжёлых production-паттернов там, где это не требуется для данного объёма нагрузки.

## Архитектура

Решение разделено на несколько логических частей:

- **Domain** — базовые модели и контракты;
- **Application** — pipeline обработки, дедупликация, метрики, фоновые сервисы;
- **Infrastructure** — адаптеры бирж, WebSocket-коннекторы, PostgreSQL repository, инициализация схемы;
- **Simulator** — локальный WebSocket-сервер, который имитирует несколько бирж с разными форматами сообщений;
- **UnitTests** — модульные тесты;
- **IntegrationTests** — интеграционные тесты с реальной PostgreSQL через Testcontainers.

## Почему добавлены симуляторы бирж

В проект добавлены локальные симуляторы бирж, которые отправляют сообщения в **разных форматах**:

- **ExchangeA** — JSON
- **ExchangeB** — CSV
- **ExchangeC** — pipe-delimited формат

Это сделано для того, чтобы:
- упростить локальную проверку pipeline;
- иметь полностью воспроизводимый сценарий без зависимости от внешних сервисов;
- показать работу адаптеров и нормализации на разных входных форматах;
- тестировать reconnect/retry-логику в более контролируемой среде.

## Live feeds

Кроме симуляторов, в проект добавлены live-источники:

- **Binance**
- **Coinbase**

Они позволяют показать, как та же архитектура работает уже не только с локальными симуляторами, но и с реальными внешними источниками.

## Текущее состояние конфигурации

В `appsettings.json` по умолчанию:
- **Binance** — включён
- **Coinbase** — включён
- **ExchangeA / ExchangeB / ExchangeC** — выключены

При необходимости можно переключить режим работы, изменив флаг `Enabled` у нужных источников.

## Как запустить проект

### Требования

- .NET SDK 8
- Docker / Docker Compose

### 1. Поднять PostgreSQL

```bash
docker compose up -d
```

### 2. Запустить агрегатор

```bash
dotnet run --project src/TickDataAggregator.Host
```

### 3. (Опционально) Запустить симулятор бирж

Если хотите использовать локальные симуляторы вместо live-источников, сначала запустите симулятор:

```bash
dotnet run --project src/TickDataAggregator.ExchangeSimulator
```

Симулятор поднимет WS-сервер на `http://localhost:5100` с тремя эндпоинтами:

- `/exchange-a` — JSON
- `/exchange-b` — CSV
- `/exchange-c` — pipe-delimited

Затем в `appsettings.json` переключите `Enabled: true` у нужных симуляторов и `Enabled: false` у Binance/Coinbase.

### 4. Запустить тесты

```bash
dotnet test TickDataAggregator.sln
```

Интеграционные тесты требуют запущенного Docker (Testcontainers поднимает PostgreSQL контейнер автоматически).
